# Learning Roadmap

Tailored to a Mechatronics Engineering Honours graduate aiming to become a
complete applied engineering package.

## Transcript-driven priorities

**Strengths to preserve (use, don't relearn):** instrumentation,
electromechanical systems, signals and systems, data communication, dynamics,
product development, research methodology. These map to the *preserve*-badged
domains in the skills matrix — keep them warm through the Build mini-projects.

**High-priority refresh:** embedded programming, AI for autonomous systems,
applied control, project execution, professional practice, engineering
documentation, plus credited fundamentals. These are the *refresh*-badged
domains and the default recommendation source on the dashboard.

## Suggested 12-week plan (one sprint per week, dashboard checklist each week)

| Weeks | Focus | Target outcome |
|---|---|---|
| 1–2 | Electrical & Electronics Lab + Mechanical Lab | Fundamentals warm; divider/filter/ADC and second-order intuition rebuilt |
| 3–4 | PID Control Lab (all challenges) | Tuning vocabulary and metrics fluency; controls evidence pack |
| 5–6 | Embedded Systems Lab + hardware Build project | FSM + debounce + latency reasoning proven on a real board |
| 7–8 | PLC & SCADA Lab + Practice Lab FAT/FMEA | Interlock thinking and acceptance-test discipline |
| 9–10 | Robotics Lab (manual → waypoints → A*) | Odometry honesty; planning + control integration |
| 11 | AI/ML Lab | Honest evaluation habits; anomaly/PdM mini report |
| 12 | Practice Lab consolidation | Portfolio pack: traceability, FMEA, risk register, decision log |

## How to work each module

1. Read **Learn** (10 min), then go straight to **Simulate**.
2. Attempt each **Challenge**; verify the pass criteria yourself before
   marking it passed — the app trusts you, your portfolio depends on honesty.
3. Read **Diagnose** and try to reproduce at least one fault deliberately.
4. Do the **Build** project when the domain is a priority refresh area.
5. Tick **Evidence** items only when the artefact actually exists somewhere.
6. Write the **Reflect** answer — two sentences beats zero.
7. Update the **Skills Matrix** rating with an evidence link.

## Portfolio end-state

By the end of the roadmap you should have: tuned-loop evidence with metrics,
a hardware embedded build, interlock test records, planned-vs-actual robot
paths, an ML evaluation mini-report with a limitations section, and a full
professional-practice artefact pack (traceability, FMEA, risk register, FAT,
decision log) — each exported as JSON/screenshots and linked from the matrix.
